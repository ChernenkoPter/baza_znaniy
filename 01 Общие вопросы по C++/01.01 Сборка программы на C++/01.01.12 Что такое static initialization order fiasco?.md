Если в одной единице трансляции переменная объявляется (extern, статическое поле класса или структуры), потом используется для инициализации другой переменной, а определяется в другой единице трансляции.
При запуске программы глобальные переменные и статические поля классов и структур могут быть определены позже момента, когда их значение было использовано для инициализации какой-то переменной из другой единицы трансляции, т.к. порядок инициализации не определен.
Пример:
```
/* a.cpp */ extern int x; int y = x + 1;

/* b.cpp */ struct z{ static int zzz; }; int w = z::zzz;

/* c.cpp */ int x = 1; int z::zzz = 2; // эти определения могут произойти позже определений y и w
```
Можно не допускать эту проблему, если пользоваться идиомой **Construct On First Use** - заменить глобальный объект функцией, которая возвращает ссылку на локальный статический объект `A& get_a() { static A a; return a; }`
Модули (20 стандарт) также устраняют данную проблему, если верить cppreference
